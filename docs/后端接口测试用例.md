# 后端接口测试用例

> 基础路径：`http://localhost:8080`
>
> 通用响应结构：
> ```json
> {
>   "code": 200,
>   "message": "success",
>   "data": {}
> }
> ```
>
> 错误码说明：
> | 错误码 | 含义 |
> |-------|------|
> | 200 | 成功 |
> | 400 | 参数错误 / 参数校验失败 |
> | 404 | 资源不存在 |
> | 500 | 服务器内部错误 |

---

## 一、分类模块 `/api/categories`

### 1.1 获取分类列表

| 项目 | 内容 |
|------|------|
| 接口 | `GET /api/categories` |
| 描述 | 获取全部分类列表，按 sort 字段升序排列 |
| 请求参数 | 无 |

#### TC-CATE-LIST-001 正常获取分类列表

- **前置条件**：数据库中存在分类数据
- **请求**：
  ```
  GET /api/categories
  ```
- **预期响应**：
  - HTTP 状态码：`200`
  - `code`：`200`
  - `data` 为数组，每个元素包含 `id`、`name`、`icon`、`sort`、`createTime` 字段
  - 数组按 `sort` 字段升序排列
  ```json
  {
    "code": 200,
    "message": "success",
    "data": [
      {
        "id": 1,
        "name": "美食",
        "icon": "icon-url",
        "sort": 1,
        "createTime": "2026-01-01T00:00:00"
      }
    ]
  }
  ```

#### TC-CATE-LIST-002 分类表为空时返回空数组

- **前置条件**：数据库中无分类数据（或全部逻辑删除）
- **请求**：
  ```
  GET /api/categories
  ```
- **预期响应**：
  - HTTP 状态码：`200`
  - `data` 为空数组 `[]`

### 1.2 根据ID获取分类详情

| 项目 | 内容 |
|------|------|
| 接口 | `GET /api/categories/{id}` |
| 描述 | 根据分类ID获取分类详情 |
| 路径参数 | `id`（Long，必填）- 分类ID |

  ```json
  {
    "code": 200,
    "message": "success",

      "id": 1,
      "name": "美食",
      "icon": "icon-url",
      "sort": 1,
      "createTime": "2026-01-01T00:00:00"
    }
  }
  ```

#### TC-CATE-GET-002 分类不存在时返回404

- **前置条件**：数据库中不存在 ID 为 999999 的分类

  ```

### 1.3 创建分类

| 项目 | 内容 |
|------|------|
| 接口 | `POST /api/categories` |
| 描述 | 创建新分类 |
| Content-Type | `application/json` |
| 请求体字段 | 见下表 |

| 字段 | 类型 | 必填 | 校验规则 | 说明 |
|------|------|------|----------|------|

#### TC-CATE-CREATE-001 正常创建分类（仅必填字段）

- **请求**：
  ```
  POST /api/categories
  Content-Type: application/json

    "name": "新分类"
  }
  ```

  - `code`：`200`

  {
    "code": 200,
    "message": "success",
    "data": 4

### 2.3 搜索店铺（分页）

| 项目 | 内容 |
|------|------|
| 接口 | `GET /api/shops/search` |
| 描述 | 多条件搜索店铺，支持分页和排序 |
| 查询参数 | 见下表 |

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| keyword | String | 否 | - | 关键词，模糊匹配店铺名称 |
| categoryId | Long | 否 | - | 分类ID，精确匹配 |
| name | String | 否 | - | 店铺名称，模糊匹配 |
| minPrice | BigDecimal | 否 | - | 最低均价 |
| maxPrice | BigDecimal | 否 | - | 最高均价 |
| minRating | BigDecimal | 否 | - | 最低评分 |
| maxRating | BigDecimal | 否 | - | 最高评分 |
| address | String | 否 | - | 地址，模糊匹配 |
| sortBy | String | 否 | - | 排序字段（仅支持 `rating`） |
| sortOrder | String | 否 | - | 排序方向（`asc` / `desc`） |
| page | Integer | 否 | 1 | 页码 |
| pageSize | Integer | 否 | 10 | 每页条数 |

#### TC-SHOP-SEARCH-001 无条件搜索（默认分页）

- **请求**：
  ```
  GET /api/shops/search
  ```
- **预期响应**：
  - `code`：`200`
  - `data` 包含 `list`、`total`、`page`、`pageSize` 字段
  - `page` 等于 1，`pageSize` 等于 10
  ```json
  {
    "code": 200,
    "message": "success",
    "data": {
      "list": [...],
      "total": 50,
      "page": 1,
      "pageSize": 10
    }
  }
  ```

#### TC-SHOP-SEARCH-002 按关键词搜索

- **请求**：
  ```
  GET /api/shops/search?keyword=火锅
  ```
- **预期响应**：
  - `code`：`200`
  - `data.list` 中所有店铺的 `name` 包含"火锅"

#### TC-SHOP-SEARCH-003 按分类ID搜索

- **请求**：
  ```
  GET /api/shops/search?categoryId=1

- **预期响应**：

#### TC-SHOP-SEARCH-004 按价格区间搜索

  ```
- **预期响应**：
  - `code`：`200`
  - `data.list` 中所有店铺的 `avgPrice` 在 [50, 200] 区间内

#### TC-SHOP-SEARCH-005 按评分区间搜索

- **请求**：
  ```
  GET /api/shops/search?minRating=4.0&maxRating=5.0
  ```
- **预期响应**：
  - `code`：`200`
  - `data.list` 中所有店铺的 `rating` 在 [4.0, 5.0] 区间内

#### TC-SHOP-SEARCH-006 按地址模糊搜索

## 四、通用异常场景

### 4.1 HTTP方法不匹配

#### TC-GEN-METHOD-001 对GET接口使用POST方法

- **请求**：
  ```
  POST /api/categories
  ```
  （不带请求体或 Content-Type 不正确）
- **预期响应**：
  - HTTP 状态码：`415` 或 `400`

#### TC-GEN-METHOD-002 对POST接口使用GET方法

- **请求**：
  ```
  GET /api/categories
  ```
  （本应 POST 的请求用了 GET，但 GET /api/categories 是列表接口，不会报错，此场景需注意路由冲突）

### 4.2 路径参数类型错误

#### TC-GEN-PATH-001 路径参数为非数字类型

- **请求**：
  ```
  GET /api/shops/abc
  ```
- **预期响应**：
  - HTTP 状态码：`400`（类型转换失败）

### 4.3 Content-Type错误

#### TC-GEN-CTYPE-001 POST请求使用错误的Content-Type

- **请求**：
  ```
  POST /api/categories
  Content-Type: text/plain

  {"name": "测试"}
  ```
- **预期响应**：
  - HTTP 状态码：`415`

### 4.4 请求体格式错误

#### TC-GEN-BODY-001 请求体为非法JSON

- **请求**：
  ```
  POST /api/categories
  Content-Type: application/json

  {invalid json}
  ```
- **预期响应**：
  - HTTP 状态码：`400`
