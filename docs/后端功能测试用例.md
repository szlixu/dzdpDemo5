# 后端功能测试用例

> 基础路径：`http://localhost:8080`
>
> 通用响应结构：
> ```json
> {
>   "code": 200,
>   "message": "success",
>   "data": {}
> }
> ```
>
> 错误码说明：
> | 错误码 | 含义 |
> |-------|------|
> | 200 | 成功 |
> | 400 | 参数错误 / 参数校验失败 |
> | 404 | 资源不存在 |
> | 500 | 服务器内部错误 |

---

## 一、分类模块 `/api/categories`

### 1.1 获取分类列表

| 项目 | 内容 |
|------|------|
| 接口 | `GET /api/categories` |
| 描述 | 获取全部分类列表，按 sort 字段升序排列 |
| 请求参数 | 无 |

#### TC-CATE-LIST-001 正常获取分类列表

- **前置条件**：数据库中存在分类数据（初始化脚本已插入 5 条）
- **请求**：
  ```
  GET /api/categories
  ```
- **预期响应**：
  - HTTP 状态码：`200`
  - `code`：`200`
  - `data` 为数组，每个元素包含 `id`、`name`、`icon`、`sort`、`createTime` 字段
  - 数组按 `sort` 字段升序排列（美食→休闲娱乐→酒店住宿→丽人美发→生活服务）
  ```json
  {
    "code": 200,
    "message": "success",
    "data": [
      { "id": 1, "name": "美食", "icon": "/icons/food.png", "sort": 1 },
      { "id": 2, "name": "休闲娱乐", "icon": "/icons/entertainment.png", "sort": 2 },
      ...
    ]
  }
  ```

#### TC-CATE-LIST-002 分类表为空时返回空数组

- **前置条件**：数据库中无分类数据（或全部逻辑删除 is_deleted=1）
- **请求**：
  ```
  GET /api/categories
  ```
- **预期响应**：
  - HTTP 状态码：`200`
  - `data` 为空数组 `[]`

### 1.3 创建分类

| 描述 | 创建新分类 |

| name | String | 是 | @NotBlank | 分类名称，不可为空串或纯空格 |

| sort | Integer | 否 | - | 排序序号 |

  ```json

### 2.3 搜索店铺（分页）

| 项目 | 内容 |
|------|------|
| 接口 | `GET /api/shops/search` |
| 描述 | 多条件组合搜索，支持分页和排序 |

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| keyword | String | 否 | - | 关键词模糊匹配 name |
| categoryId | Long | 否 | - | 分类ID精确匹配 |
| name | String | 否 | - | 名称模糊匹配 |
| minPrice | BigDecimal | 否 | - | 最低均价 |
| maxPrice | BigDecimal | 否 | - | 最高均价 |
| minRating | BigDecimal | 否 | - | 最低评分 |
| maxRating | BigDecimal | 否 | - | 最高评分 |
| address | String | 否 | - | 地址模糊匹配 |
| sortBy | String | 否 | - | 排序字段（仅支持 rating） |
| sortOrder | String | 否 | - | asc / desc |
| page | Integer | 否 | 1 | 页码 |
| pageSize | Integer | 否 | 10 | 每页条数 |

#### TC-SHOP-SEARCH-001 无条件默认分页查询

- **请求**：
  ```
  GET /api/shops/search
  ```
- **预期响应**：
  - `data` 包含 `list`、`total`、`page`=1、`pageSize`=10
  - `total` = 数据库中未删除的店铺总数（初始 10 条）

#### TC-SHOP-SEARCH-002 按关键词搜索

- **请求**：
  ```
  GET /api/shops/search?keyword=火锅
  ```
- **预期响应**：
  - `data.list` 中所有店铺 name 包含"火锅"
  - 匹配结果：海底捞火锅(中关村店)

#### TC-SHOP-SEARCH-003 按分类ID筛选

- **请求**：
  ```
  GET /api/shops/search?categoryId=2
  ```
- **预期响应**：
  - 所有结果 categoryId = 2（休闲娱乐：万达影城、大玩家超乐场）

#### TC-SHOP-SEARCH-004 按价格区间筛选

- **请求**：
  ```
  GET /api/shops/search?minPrice=50&maxPrice=150
  ```
- **预期响应**：
  - 所有结果 avgPrice ∈ [50, 150]

#### TC-SHOP-SEARCH-005 按评分区间筛选

- **请求**：
  ```
  GET /api/shops/search?minRating=4.5&maxRating=5.0
  ```
- **预期响应**：
  - 所有结果 rating ∈ [4.5, 5.0]
  - 匹配：海底捞(4.8)、西贝莜面村(4.7)、全聚德(4.6)

#### TC-SHOP-SEARCH-006 按地址模糊搜索

- **请求**：
  ```
  GET /api/shops/search?address=朝阳
  ```
- **预期响应**：
  - 所有结果 address 包含"朝阳"
  - 匹配：万达影城、大玩家超乐场、如家酒店、文峰美发、链家地产、西贝莜面村

#### TC-SHOP-SEARCH-007 按名称模糊搜索

- **请求**：
  ```
  GET /api/shops/search?name=烤鸭
  ```
- **预期响应**：

#### TC-SHOP-SEARCH-008 按评分降序排列

  ```
  GET /api/shops/search?sortBy=rating&sortOrder=desc
  ```
